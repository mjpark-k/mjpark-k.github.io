## 비동기 자바스크립트

자바스크립트의 꽃이다.

### 비동기가 무엇인가
- 맥도날드에서 매니저가 있으면 ,버거를 직접만들지 않음
- 주문을 직원들에게 나누어주고 동시에 처리할 수 있도록 함.
- 라면 끓이는 동안 다른 일들을 할 수 있다.
- 이것을 언어로 표현한 것임.
```js
boilRamenWater(() => {
  console.log("boiling!!");
})

### 가장 간단한 비동기 함수 `setTimeout`
- 비동기 프로그래밍에 콜백이 매우 자주 쓰임

- 기다리는 동안에 다른 일을 하게 됨
- 이것이 비동기 프로그래밍의 큰 특징
- **동시에 작업하는 것과는 차이가 있음!**
- **기다리는 동안 다른 일을 하는 것임!**

### 비동기 함수의 실행 순서
- **순서를 명시한 것이 아니면 순서를 생각하지 않는 것이 좋다**
- 단순히, 지정한 일이 일어나면 실행된다고 생각하는게 편함
- setTimeout ms를 0으로 해도 바로 실행되지 않음
- 오히려 다른 모든 것이 실행되고 실행됨
- 만약에 1000으로 하면 정확히 1초 후에 실행될까??
  - 그렇지 않다! 다른 작업들을 처리하고 1초 이상 후 실행된다.
- 그러면 언제쯤 실행될지를 영원히 알 수 없는가?
- 다른 일들이 껴 있어서 그런것임. 다른 일이 너무 오래 걸리지 않는다면 금방 실행될 것
- 실행되어야 하는 콜백이 여러 개가 쌓이면, 쌓이는 순차적으로 실행됨

### 비동기 함수의 예씨들
- `htmlElement.addEventListener`
- `setTimeout`
- `setInterval`
- `fs.readFile(callback, ...)` -> node.js

### 콜백 헬
- 비동기 함수에 콜백을 제공해서 연속된 일을 처리하다보면 코드읽기가 어려워짐

## Promise란?

### `fetch` 정의
- URL로 서버에 요청을 보내서, 데이터를 받아오는 것
- 기다리고 값을 리턴하는 것이 아니라 Promise를 리턴함
- await

### `Promise`의 정의
- 어떤 값을 나중에 돌려주겠다는 약속
- console.log로 찍어보면, pending이라고 나옴
- 이를 await로 값을 받을때까지 기다릴 수 있음
- await는 기다려야 하는 값을 사용하고 싶으니, 기다리겠다고 하는 것
